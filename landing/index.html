About

AI that stays
on your machine

OllamaBro was built for developers and power users who want the full capabilities of modern AI without sending their conversations to cloud providers. Every message, every model response, every conversation history lives entirely on your local hardware.

The extension communicates with a tiny local proxy server ‚Äî a lightweight Node.js/Express app that bridges Chrome's extension sandbox with Ollama's local API. It handles CORS, enforces localhost-only connections, and automatically enables streaming.

No account. No subscription. No telemetry. No API keys required for core chat. Just you, your models, and your data.

MIT License Zero Telemetry No Account Needed Manifest V3 Open Source

Privacy by Design

Your conversations never leave your machine. No cloud API keys, no remote storage, no third-party access to your data ‚Äî ever.

Fully Open Source

Every line of code is on GitHub. Audit it, fork it, extend it, or contribute. The community drives the roadmap.

Runs Anywhere Ollama Does

If Ollama runs on your hardware, OllamaBro works. Any OS, any GPU (or no GPU), any model in the Ollama library.

Verify Ollama is running: Navigate to http://localhost:11434 in your browser. You should see Ollama is running. If not, start it with ollama serve.

Install the Proxy Server

A lightweight Express app bridges Chrome's sandbox with Ollama's API, handling CORS, request filtering, and web context injection.

1
Clone the repository
git clone https://github.com/BorisHrzenjak/OllamaBro
cd OllamaBro
2
Install dependencies
cd proxy_server
npm install
3
Configure environment (optional)
Create a .env file for your Tavily API key to enable web search.
# proxy_server/.env
TAVILY_API_KEY=your_tavily_key_here
4
Start the proxy server
# Direct start
npm start

# PM2 ‚Äî persists across reboots (recommended)
pm2 start server.js --name ollama-proxy
pm2 save && pm2 startup

The proxy listens on port 3000, accepts only localhost connections, and proxies only /api/tags, /api/chat, /api/generate, /api/show, and /api/pull. Nothing else passes through.

Model Parameters

Fine-tune every generation with six configurable parameters. All parameters are per-model and persist across sessions. Find them in Settings ‚Üí Model Parameters.

‚ú¶
Temperature (0.0 ‚Äì 2.0, default 0.8)
Controls randomness. Lower values (e.g. 0.2) make output more deterministic and factual. Higher values (e.g. 1.4) make it more creative and varied.
‚ú¶
Top P (0.0 ‚Äì 1.0, default 0.9)
Nucleus sampling threshold. The model only samples from the top tokens whose cumulative probability reaches this value. Lower = more focused output.
‚ú¶
Top K (integer, default 40)
Limits the sampling pool to the top K most probable tokens. Reduces incoherence without sacrificing too much variety.
‚ú¶
Repeat Penalty (0.0 ‚Äì 2.0, default 1.1)
Penalises recently generated tokens to discourage repetition. Increase if the model loops on the same phrases.
‚ú¶
Seed (integer)
Set a fixed seed for fully reproducible outputs. Useful for testing or sharing exact responses. Leave blank for random seeding.
‚ú¶
Num Predict (integer)
Maximum tokens the model will generate per response. Cap this to keep responses concise or budget context space for long conversations.

Each parameter has a paired slider and number input that stay in sync. Click Reset to clear all overrides and return to model defaults.

Deep Research

Deep Research uses Tavily's multi-step research API to pull from many sources, synthesise findings, and inject a full cited report as context ‚Äî far more comprehensive than basic web search.

‚ú¶
Activate with the brain icon
Click the üî¨ brain icon in the input bar (next to the web search globe). Deep Research and Web Search are mutually exclusive ‚Äî enabling one disables the other.
‚ú¶
How it works
The proxy calls Tavily's Research API with max depth 3 and breadth 4, traversing multiple search layers. The resulting report ‚Äî including source citations ‚Äî is prepended to the model's context as "Deep Research Report:".
‚ú¶
Best for
Complex research questions, literature overviews, comparative analyses, or any query where you need depth rather than a quick answer. Expect a few extra seconds of latency while the research runs.

Text-to-Speech

OllamaBro supports two TTS engines: the browser's built-in Web Speech API and the high-quality local Kokoro TTS model (runs entirely in-browser via ONNX).

‚ú¶
Kokoro TTS (recommended)
A ~86 MB ONNX model loaded on first use. Offers 28 voice profiles (14 female, 14 male) in English US and GB variants, each graded A+ to F for quality. Generates audio sentence-by-sentence for fast streaming playback.
‚ú¶
Web Speech API
Uses Chrome's built-in voices ‚Äî no download required. Voices are grouped by language. Quality varies by OS and Chrome version. Good fallback if you don't want to load Kokoro.
‚ú¶
Using TTS
Click the speaker üîä icon on any assistant message to read it aloud. Or press CtrlR to read the last response. A stop button appears during playback. Markdown, code blocks, and <think> tags are stripped before synthesis.
‚ú¶
Setup
Open Settings ‚öôÔ∏è ‚Üí TTS Settings. Select your engine and voice. For Kokoro, click Pre-load Model to warm it up before your first use ‚Äî subsequent uses load instantly from cache.

LLMFit Integration

LLMFit is a separate CLI tool that analyses your hardware and recommends Ollama models that will fit in your VRAM/RAM. OllamaBro integrates directly with it.

1
Install LLMFit
Install the llmfit CLI on your system. OllamaBro calls it via llmfit --json fit ‚Äî it must be on your system PATH.
2
Open Recommended for Your Hardware
Click the CPU icon or open Settings ‚Üí Hardware Recommendations. OllamaBro spawns llmfit with a 30-second timeout and parses the JSON output.
3
Browse recommendations
Model cards display: name & organisation, estimated tokens/sec, VRAM usage %, best quantization, run mode (CPU/GPU), parameter count, category (code / chat / reasoning), and a 0‚Äì100 fit score bar. Filter by fit level: Perfect, Good, Marginal, Too Tight.
4
Pull recommended models
Each card has a Pull button. Click it and OllamaBro auto-fills the model name in Model Management and starts the download. HuggingFace model names are automatically converted to Ollama pull names.

If llmfit is not installed, the panel gracefully shows "llmfit not available" with a link to install it. Core chat features are completely unaffected.
